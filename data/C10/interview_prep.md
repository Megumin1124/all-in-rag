# Redis 为什么快
**标签**: Redis, 缓存, 数据结构, IO多路复用
**公司**: 字节跳动
**问题**: Redis 为什么快？
**回答**: Redis 性能高通常来自三点：第一，基于内存读写，避免了磁盘随机 IO；第二，核心命令执行路径短，数据结构针对高频场景做了优化；第三，单线程事件循环结合 IO 多路复用，减少线程切换和锁竞争。在回答时最好顺带补充：Redis 也并非绝对单线程，网络读写与部分后台任务会用到额外线程。

# MySQL 索引失效场景
**标签**: MySQL, 索引, SQL优化
**公司**: 美团
**问题**: 什么时候会发生 MySQL 索引失效？
**回答**: 常见场景包括对索引列做函数运算、违反最左前缀原则、隐式类型转换、范围查询后继续使用联合索引后缀列、以及模糊匹配以 `%` 开头。回答时建议补充 explain、覆盖索引、回表成本等工程化观察。

# 如何设计限流系统
**标签**: 系统设计, 限流, Redis
**公司**: 阿里
**问题**: 如果让你设计一个限流系统，你会怎么做？
**回答**: 先说明目标：保护下游、维持稳定性、可观测。再分层设计：网关层 + 服务层双层限流；算法上比较固定窗口、滑动窗口、令牌桶、漏桶；工程上强调热点 key、时钟漂移、分布式一致性与降级策略。若使用 Redis，可以提到 Lua 保证原子性。
